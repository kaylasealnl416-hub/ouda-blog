# Incident Spec: 首页不可访问修复

> 状态：执行中
> 日期：2026-04-20
> 分支：`codex/fix-homepage-outage`

## 1. 现象

- 线上根路径 `/` 无法正常访问
- 后台路径 `/admin` 可正常打开和使用
- 故障影响公开访客访问首页与首页引流

## 2. 根因假设

- 首页与文章详情页对 `category` 使用了强假设，渲染时直接读取 `CATEGORY_META[post.category].emoji`
- 写接口 `POST /api/posts` 与 `PUT /api/posts/:id` 未做服务端校验，异常 `category/tags/createdAt` 可直接写入数据库
- 一旦某篇已发布文章写入异常分类，首页重新渲染或缓存失效后会触发运行时崩溃

## 3. 修复目标

- 首页遇到脏数据时不崩溃，公开页面恢复可访问
- 写接口拒绝非法分类、非法标签和非法日期，阻断脏数据继续进入数据库
- 增加回归测试，覆盖首页容错与 API 校验

## 4. 非目标

- 不在本次修复中重构 CMS 表单
- 不在本次修复中调整 UI 视觉
- 不做数据库批量清洗脚本，先通过运行时容错恢复线上

## 5. 验收标准

- 公开首页在存在异常 `category` 的文章数据下仍能正常渲染
- 文章详情页在异常 `category` 下仍能正常渲染
- `POST /api/posts` 和 `PUT /api/posts/:id` 对非法 `category` 返回 `400`
- 标签非数组、标签元素为空串、非法日期返回 `400`
- 现有关键测试继续通过
